*Risk 2, API troubles:*
*Scott did state that the API has no documentation as well, so this could be a problem in integrating.*
*A plan to remedy this is; as we acquaint ourselves with the project and its tools, either ask Scott for documentation,*

Rewriting the old code to fit the needs of v2, which is just going to be replaced, was not something we really saw as a great option, so we decided to focus on the other goals which Scott viewed just as important.

As for Risk 2, this was not as bad as we thought, and mostly requires work on Scott's end. Therefore, we only really need to get it to a state where Scott only has to connect the pieces.
